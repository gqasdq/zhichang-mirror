# 职场镜子 Career Mirror

## 智联招聘首届全国AI创新大赛 · AI+求职赛道作品说明文档

***

## 一、作品简介

"职场镜子"，定位是求职路上的"深夜好朋友"。

作为GIS专业的应届生，我在秋招期间投递了上百份简历，经历了十余次面试，最终意识到一个核心问题：**比找不到工作更让人焦虑的，是不清楚自己适合什么方向。** 智联招聘调研数据显示，36.2%的未就业毕业生将失败归因于"过度迷茫，不清楚自身适配方向"——我本人就是这36.2%中的一员。

我尝试过市面上大部分求职工具：简历优化工具、AI面试助手、职业测评平台……但它们存在一个共同的局限——**只解决表象问题，无法缓解根本焦虑。** 简历写得再好，内心的迷茫并不会减少；面试技巧背得再熟，对自己的方向依然没有底气。

因此，我决定做一款从情绪疏导到行动建议的完整链路产品。它不是替你做决定，而是帮你**看见自己**。

职场镜子目前有6个功能模块：

- **情绪急救站**：让你把憋在心里的话说出来的树洞
- **金子探测器**：告诉你"你的经历比你以为的更有价值"
- **平行宇宙**：帮你看清不同选择可能会带来什么
- **职业基因**：挖掘你自己都没意识到的优势
- **人才共情链**：找到跟你走过同样路的同行者
- **首页**：一个情绪温度计，先承认你很难，再带你往前走

该项目已完成开发，技术方案验证通过，所有功能模块可正常运行，数据层已填充真实内容。下文将从市场背景、方案设计、技术实现、商业规划等方面进行详细说明。

## 二、市场调研

### 2.1 求职市场现状

根据教育部及多家机构公开数据：

2026届高校毕业生1270万人，同比增加48万，连续5年增长。加上往届待业、海归回流，全年求职总规模超过1500万。但面向应届生的有效校招岗位只有567万个左右，供需比约2.7:1，接近3人竞争1个岗位。

专业间分化严重。文科就业率73.5%，显著低于理科86%。汉语言、法学等专业供需比达7:1，而AI/大模型/算法方向供需比仅为1:4，应届硕总包可达30万以上。

16-24岁青年失业率16.1%-17.3%，中国青年网调查数据显示近9成大学生对就业前景感到担忧。

另一个值得关注的趋势：考研报名人数暴跌131万至343万，而国考报名371.8万首次超过考研，25.1%毕业生首选考公考编。从"卷学历"转向"卷编制"，反映出年轻群体对确定性的强烈渴求。

### 2.2 AI工具使用普遍，但焦虑并未缓解

前程无忧《毕业生求职行为与就业心理调查》显示，超过86%的2026届毕业生在求职中使用AI工具，但仅17%完全依赖AI生成内容，大多数人将其定位为辅助工具。

关键问题在于，AI辅助完成后焦虑反而加重。35%的毕业生使用AI优化简历后产生焦虑，担心"面试表现无法匹配简历水平"——简历层面的提升，并未解决能力自信的问题。

与此同时，25%的初级白领岗位面临AI替代风险，企业招聘明确要求"会用AI工具"的岗位同比增长99.21%。工具能力的普及反而加剧了新的焦虑：大家都在用AI，差异化优势从何而来？

### 2.3 现有产品的不足

经过对市面上主流AI求职产品的调研，现有方案可分为以下几类：

**企业端招聘工具**（北森、Moka、用友大易等）——这类产品从企业视角出发，帮助HR筛选简历和评估候选人。对求职者而言，这些工具是筛选机制的组成部分，而非助力。

**求职辅导机构**（职比特、青松求职等）——收费从几千到5万不等，普通学生难以负担。且服务质量参差不齐，行业缺乏统一标准。

**通用AI工具**（ChatGPT、Kimi等）——能完成简历润色、面试模拟等基础任务，但缺乏专业领域知识。对于细分专业的简历分析，通用模型的输出往往流于表面。

**简历优化工具**（各类AI简历生成器）——仅停留在文案层面的优化，无法改变求职者对自身价值的认知。

综合来看，**目前没有一款产品真正站在求职者角度，系统性地解决"迷茫"这个核心痛点。**

### 2.4 市场空白与产品机会

基于以上分析，当前市场存在一个明显的空白地带：**情绪疏导 + 自我认知 + 行动建议**的完整链路产品。

现有产品要么只做工具（改简历、搜岗位），要么只做培训（面试课、职场课），要么只做测评（MBTI、职业测评），但没有一款产品将情绪和工具串联起来。

而实际观察到的用户需求是：求职者的核心痛点不是"不会写简历"，而是"不知道自己能干什么"。这个认知问题不解决，简历改100遍也无法从根本上缓解焦虑。

## 三、方案介绍

### 3.1 产品定位

**求职路上的"深夜好朋友"——情绪疏导 + 自我认知 + 行动建议的完整链路。**

核心原则：AI不是替代人的决策者，而是陪伴人的发现者——帮你看见，不替你决定。

这个定位源于我个人的求职体验：最难熬的时刻不是海投无果，而是没有人理解你为什么这么难。我需要的是先被理解，然后一起寻找答案。

### 3.2 设计哲学

在设计职场镜子时，我坚持以下几个原则：

**先处理情绪，再处理问题。** 多数产品一上来就问"想找什么工作"，但在回答这个问题之前，用户需要先被理解。因此情绪急救站是核心入口，而非附赠功能。

**不给你答案，帮你看见选项。** 平行宇宙不是算命工具，不会告诉你"选A一定好"，而是把A和B的可能结果都呈现出来，让用户自己做判断。

**你的经历比你以为的更有价值。** 金子探测器的核心理念。许多求职者并非没有经历，而是无法识别自身经历中的可迁移价值。普通的实习和项目经历同样蕴含竞争优势，关键在于如何被识别和表达。

**你不是一个人。** 人才共情链的存在意义。每位求职者都可能觉得自己是唯一遇到困境的人，但实际上有大量前辈走过同样的路。

### 3.3 产品架构

整体架构分为5层：

**表现层**：Streamlit前端，暖色系"深夜客厅"设计语言，主背景色#F7F3EF，整体视觉风格温暖柔和，避免工具类产品的冰冷感。

**功能层**：6大功能模块，各司其职，又互相连接。

**引擎层**：6个专业化Agent，每个Agent拥有独立的Prompt工程，包含1500-2200行专业指令。

**知识层**：FAISS向量数据库 + BGE-large-zh-v1.5嵌入模型，职业基因知识库2221行，职场故事库803条。

**模型层**：DeepSeek V3（主力推理）+ 智谱GLM-4-Flash（轻量任务），双模型协作，月运营成本低于5元。

## 四、核心功能及创新点

### 4.1 情绪急救站

**功能描述**：单轮倾诉式情绪陪伴。用户选择一个情绪标签，说出最堵在心里的话，AI给予共情式回应。

**使用场景**：深夜失眠，盯着求职APP不知如何继续，想找人倾诉但不想打扰朋友。

**设计思路**：求职焦虑不敢说、无处说，越憋越崩溃。这个模块不提供职场建议、不灌鸡汤，核心目标是让用户感受到**被听见了**。

技术方案用 DeepSeek V3 + 共情式 Prompt 工程，单轮问答模式。Prompt 里明确要求"不评判、不建议、不安慰"，只回应情绪。

**创新点**：将情绪处理置于求职工具的入口位置，这在现有产品中尚无先例。市面产品都在教你怎么写简历、怎么面试，但没有人先关注"你今天还好吗"。

### 4.2 金子探测器

**功能描述**：4-Agent 协作式简历分析。上传简历 → 核心优势提取 → 岗位差距分析 → 金子报告生成。

**使用场景**：简历写了十几版，越改越迷茫，不知道自己的经历到底值不值一提。

**设计思路**：告诉用户"你的经历比你以为的更有价值"。短板也能翻案。

核心技术是4个 Agent 协作：

- **分析器（analyzer）**：提取简历结构化信息，输出核心能力标签
- **匹配器（matcher）**：分析目标岗位差距，识别可迁移能力
- **报告器（reporter）**：生成完整金子报告，包含优势、差距、行动建议
- **探测器（detector）**：深挖隐藏优势，防止低估自己

每个 Agent 有独立专业 Prompt，1500-2200行，角色设定 + 输出结构约束 + 禁止规则 + 案例示范。核心优势从 analyzer 的结构化字段精准提取，不凑数不编造。

技术方案用 DeepSeek V3 + 智谱 GLM-4-Flash 双模型。为什么用两个？因为 DeepSeek 推理能力强但有并发限制，智谱免费不限量可以兜底。

**创新点**：4-Agent协作模式，每个Agent专注单一任务，组合输出比单一模型更专业的结果。核心优势提取采用结构化字段精准读取，而非让模型自由发挥，确保输出有据可依、不凑数不编造。

### 4.3 平行宇宙

**功能描述**：输入当前纠结的选择，推演不同选择的可能性。

**使用场景**："回老家考编"还是"留在大城市卷互联网"？不是替你决定，是帮你看见。

**设计思路**：站在人生岔路口不知道另一个选择会怎样。输入两个选项，AI 帮你推演每个选择可能带来什么机会、什么代价、什么心态变化。

技术方案用 DeepSeek V3 + 多维度推演 Prompt，输入选项后从职业发展、生活质量、心理状态、风险评估等多个维度展开分析。

**创新点**：将"选择推演"做成可视化工具，不是算命式的"选A最好"，而是把每个选择的正反面都呈现出来，帮助用户做出更清醒的决定。

### 4.4 职业基因

**功能描述**：基于8章节2221行知识库的深度职业基因测序。分析显性基因 + 隐藏基因 + 基因陷阱 + 推荐岗位。

**使用场景**：说不清自己擅长什么、适合干什么。做了很多测评，但还是不知道自己几斤几两。

**设计思路**：有些优势你自己不知道但别人看得见。这不是 MBTI 那种贴标签的测评，而是基于你的经历和知识库，给你一个**有血有肉**的分析。

知识库8章节包括：

1. 职业能力模型
2. 岗位画像
3. 行业趋势
4. 转型路径
5. 能力迁移
6. 性格-职业映射
7. 简历-岗位匹配
8. 职业发展周期

技术方案用 FAISS 向量检索 + BGE-large-zh-v1.5 嵌入模型 + DeepSeek V3 推理 + RAG 增强。追问 Prompt 有8章节详细模板，禁止重复已展示信息，查看详情时传 existing\_job\_info。

**创新点**：区别于通用测评工具，职业基因有专业知识库支撑，分析结果有真实数据增强。803条真实故事+2221行知识库，确保AI输出有据可查，而非凭空生成。

### 4.5 人才共情链

**功能描述**：803条真实职场故事构成的共情网络，16个处境标签（毕业即失业、专科逆袭、Gap 空白期等）。

**使用场景**：觉得自己是唯一一个这么惨的人。找到走过同样路的同行者。

**设计思路**：每个人都可能觉得自己是处境最差的那个，但事实上大量前辈经历过同样的困境。这个模块帮助用户找到共鸣，传递"你不是一个人"的信号。

技术方案用 BGE 嵌入 + FAISS 语义检索 + 智谱 GLM-4-Flash。故事来源覆盖知乎、小红书、脉脉等平台真实经历分享，每条标注出处和标签。

**创新点**：真实故事库而非AI生成内容。803条故事来自真实求职者经历，标注来源和处境标签，比AI编造的故事更具共情力和可信度。

### 4.6 首页

**功能描述**：情绪温度计 + 全功能入口。一句"求职这条路，有时候真的挺难熬的"打开整个产品。

**设计思路**：首页不是功能列表，而是一种态度——先承认求职的困难，再引导用户前行。

情绪温度计让你用滑动条选择当前情绪状态，不同状态显示不同的引导语和推荐功能。

### 4.7 隐私保护

**功能描述**：零门槛、零留存的隐私保护架构。用户无需注册登录，用完即走，关闭页面即清除所有数据。

**使用场景**：每个求职者都不希望自己的简历和情绪倾诉被泄露或被他人看到。

**设计思路**：作为"深夜好朋友"，信任是第一位的。用户愿意倾诉脆弱，前提是确定自己的信息不会被泄露。因此隐私保护不是附加功能，而是产品的基础架构。

具体实现6项措施：

1. **API调用脱敏**：所有用户输入在发送给大模型前，自动过滤手机号、邮箱、身份证号、银行卡号等个人身份信息，替换为占位符。AI看不到你的真实身份，但不影响分析质量——大模型不需要知道你的手机号就能分析你的简历。
2. **数据不落地**：用户简历原文和对话内容不写入数据库，仅存于Streamlit会话状态中。关闭浏览器后数据自动清除，服务端不保留任何个人数据。超过24小时的临时文件也会被自动清理。
3. **向量库脱敏**：用户简历生成FAISS向量前，先经过脱敏处理，确保嵌入向量中不包含任何个人身份信息。
4. **多用户隔离**：基于Streamlit匿名会话ID实现数据隔离，每个用户的数据完全独立、互不可见，无需注册登录即可使用。
5. **报告PDF导出**：分析报告可导出为PDF保存到用户本地设备，服务端不存储报告原文。用户拥有数据主权。
6. **一键清除**：侧边栏提供"清除我的所有数据"按钮，随时可删除所有会话信息。

**创新点**：将隐私保护作为产品基础架构而非附加功能，"零门槛+零留存"的设计让用户无需注册、无需信任即可使用，这在求职类产品中几乎没有先例。多数求职工具需要注册登录、上传简历并持久化存储，职场镜子反其道而行——用完即走，不留痕迹。

## 五、技术架构与AI能力

### 5.1 整体技术架构

```
用户 → Streamlit前端 → 6大功能模块 → Agent引擎层 → LLM API
                    ↓
              知识库层(FAISS)
                    ↓
              嵌入模型(BGE)
```

### 5.2 各层级技术选型

| 层级      | 技术选型                                   | 说明                                     |
| ------- | -------------------------------------- | -------------------------------------- |
| 前端      | Streamlit + 自定义CSS                     | 暖色系"深夜客厅"设计语言，主背景#F7F3EF               |
| Agent引擎 | 6个专业化Agent + Prompt工程                  | 每个模块独立Agent，每个Agent有1500-2200行专业Prompt |
| 知识库     | FAISS + BGE-large-zh-v1.5              | 职业基因知识库2221行，故事库803条                   |
| LLM     | DeepSeek V3（主力推理）+ 智谱GLM-4-Flash（轻量任务） | 双模型协作，成本极低                             |
| 存储      | SQLite + session\_state                | 会话状态管理+匿名统计，不存用户原始数据                   |
| PDF解析   | pdfplumber                             | 简历PDF上传解析                              |
| 异常处理    | 统一错误边界模块                               | 友好提示，不暴露Python报错                       |
| 隐私保护    | privacy\_filter + session\_manager     | API脱敏+数据不落地+多用户隔离+一键清除                 |

### 5.3 Agent工程细节

每个功能模块对应一个独立 Agent：

**情绪急救站 Agent**：共情式 Prompt，不评判、不建议、不安慰，只回应情绪。1500行 Prompt，核心规则是"让用户觉得被听见了"。

**金子探测器 Agent 集群**：

- analyzer（1500行）：简历结构化分析，输出核心能力标签
- matcher（1800行）：岗位匹配分析，识别差距
- reporter（2000行）：报告生成，整合分析结果
- detector（2200行）：隐藏优势挖掘，防止低估

每个 Agent 的 Prompt 都包含：

- 角色设定（你是谁，你擅长什么）
- 输出结构约束（必须包含哪些字段，格式是什么）
- 禁止规则（绝对不能说什么、做什么）
- 案例示范（好的回答长什么样）

### 5.4 知识库构建

**职业基因知识库**：2221行，8章节结构，涵盖职业能力模型、岗位画像、行业趋势、转型路径、能力迁移、性格-职业映射、简历-岗位匹配、职业发展周期。

**人才共情链故事库**：803条真实职场故事，覆盖16个处境标签。来源包括知乎、小红书、脉脉等平台真实分享，每条标注出处和适用场景。

知识库通过 BGE-large-zh-v1.5 嵌入后存储在 FAISS 向量数据库，支持语义检索。用户提问时，先在知识库中检索相关片段，再结合 DeepSeek V3 推理生成回答。

### 5.5 使用的模型

**DeepSeek V3**（主力推理）：

- 用途：复杂推理、深度分析、多步骤任务
- 优势：推理能力强，中文理解好
- 成本：¥0.2/百万Token，赠送500万Token

**智谱GLM-4-Flash**（轻量任务）：

- 用途：语义检索、故事匹配、轻量生成

双模型协作的好处是：DeepSeek 负责需要深度思考的任务，智谱负责需要高并发的任务，整体成本极低。

## 六、应用场景

### 场景一：秋招焦虑期

小王是某211高校市场营销专业应届生，秋招投了50份简历，收到3个面试，都挂在终面。他开始怀疑人生，觉得自己是不是不够努力。

打开职场镜子，先在情绪急救站选了"挫败"标签，说出心里话："我明明很努力了，为什么还是这样？"

AI没有说"加油你可以的"，而是回应："我听到了。你很努力，这很重要。但有时候努力和结果之间不是线性的，特别是秋招这种竞争激烈的时候。"

随后他进入金子探测器，上传简历后，分析报告显示他实际上具备多项可迁移能力，只是在简历中没有有效表达。他意识到问题不是能力不足，而是表达方式不当。

### 场景二：职业方向迷茫

小李是文科生，学的汉语言文学，不知道自己能干什么。考研不想考，考公竞争太大，想去互联网又觉得自己没技能。

他进入职业基因，输入自己的经历（学生会干部、公众号运营、两次实习），系统分析出他具备内容创作能力和用户洞察能力，推荐了内容运营、新媒体策划、用户运营等方向，每个方向都有详细的岗位职责和能力要求。

接着他进入平行宇宙，输入"回老家考编"vs"留大城市做运营"，系统推演出每个选择的机会、代价和心理预期变化。他看完后表示："思路清晰了很多。"

### 场景三：Gap Year 焦虑

小张因为考研失败后没有找工作，在家待了半年。现在想求职，但不知道简历上的空白期怎么解释，担心面试被问。

他进入人才共情链，选择"Gap空白期"标签，看到了许多真实的故事：有人Gap期间做了自由职业、有人Gap期间学了新技能、有人Gap期间做了志愿者……他意识到Gap不是简历污点，关键在于如何定义和利用这段时间。

随后他进入金子探测器，系统帮助他将Gap期间的自学和项目经历转化为有价值的简历内容。

### 场景四：深夜失眠

凌晨2点，小赵睡不着，盯着天花板想明天还要继续投简历。他打开职场镜子，情绪温度计显示他的状态是"疲惫"。

首页说："求职这条路，有时候真的挺难熬的。今晚不用想那么远，先让自己喘口气。"

他点进情绪急救站，选了"疲惫"标签，说："我不想投简历了，好累。"

AI回应："累是正常的。这条路本来就不好走。今晚可以不投简历，明天再试也不迟。"

那一刻，他感到被理解了。

## 八、商业化思考

### 8.1 商业化的可行性分析

很多学生项目难以持续，核心原因是缺乏可行的商业模式。职场镜子的商业化路径相对清晰，原因有三：

**第一，用户有真实付费意愿。** 求职辅导市场规模从99元AI测评到5万元私人陪跑，说明用户愿意为有效服务付费。当前市场的问题是定价过高或服务质量不稳定，而非需求不足。

**第二，存在多个潜在付费方。** 求职者本人、高校就业中心、企业HR部门都可能成为付费方。

**第三，数据具备增值空间。** 积累的用户能力图谱属于稀缺数据资产，可支撑反向招聘、人才推荐等高价值业务。

### 8.2 商业模式设计

**免费 + 增值模式（ Freemium）**：

| 功能          | 免费 | 付费      |
| ----------- | -- | ------- |
| 情绪急救站       | ✓  | ✓       |
| 人才共情链（基础版）  | ✓  | -       |
| 人才共情链（完整版）  | -  | ¥9.9/月  |
| 金子探测器（简要版）  | ✓  | -       |
| 金子探测器（完整报告） | -  | ¥19.9/次 |
| 职业基因（简要版）   | ✓  | -       |
| 职业基因（深度测序）  | -  | ¥29.9/次 |
| 平行宇宙        | ✓  | ✓       |

免费功能覆盖80%用户，付费功能服务愿意为深度服务付钱的20%用户。

**高校合作模式**：

高校就业中心有服务毕业生的 KPI，但人手有限。职场镜子可以：

- 按学校付费：每校每年1-3万
- 按使用量付费：每服务一个学生收10-30元
- 定制服务：帮学校做就业数据分析报告

**企业端延伸**：

30.6%应届生3个月内离职，企业招聘成本平均1-2万/人。如果职场镜子能帮助降低流失率，企业有明确的付费动力。

产品形态：为新员工提供入职适应陪伴，试用期跟踪辅导。按人头收费：每人每月99元，6个月试用期共594元，相比1-2万的招聘成本，投入产出比可观。

**数据增值模式**：

积累足够的用户能力图谱数据后，可以：

- 反向招聘：企业根据能力图谱主动找人，收猎头费
- 行业报告：发布就业趋势分析，收报告费
- 人才数据库：做成 B 端产品卖给企业

### 8.3 竞争优势分析

相比大公司或成熟团队，个人项目为何具备竞争力？

**第一，定位差异。** 大公司做的是B端（帮企业招人），职场镜子做的是C端（帮求职者找方向）。C端产品需要更细腻的情绪感知和陪伴体验，大公司通常不愿投入资源做这类"重体验、轻变现"的产品。

**第二，成本优势。** 项目运营成本低于5元/月，大公司的运营成本是数百倍于此。极低的成本结构意味着可以用更低价格服务用户，甚至免费提供基础功能。

**第三，迭代速度。** 一个人就是一个团队，决策链路短、迭代周期快。大公司需要走流程、等排期，个人项目可以快速响应用户反馈。

**第四，情感定位。** "深夜好朋友"的情感定位是大公司难以复制的——他们的产品是功能导向的工具，而非有温度的陪伴。

## 九、总结

职场镜子是一款面向求职者的陪伴型产品，核心出发点是我作为应届生自身的求职经历与思考。

它不完美，但它试图解决一个真实存在的问题：**求职最难的不是不会写简历、不会面试，而是不清楚自己是谁、能干什么、适合什么方向。**

产品从情绪疏导入手，陪伴用户走过最难的那段路。AI不是替代人的决策者，而是帮助人发现自身的可能。

如果有机会将职场镜子发展为一款真正的产品，我希望它能帮助更多像我一样经历过迷茫、焦虑和自我怀疑的求职者——让他们知道：你不孤单，你的经历有价值，你的选择值得被认真对待。
